## 一、Agenda：已识别的问题

### 🔴 P0 — 底层设计问题

**ISSUE-001: 零件组合深度与战斗反馈厚度不匹配**

- **症状**：864 种坦克组合，但打起来感受差异不大。组装 2 分钟，进战斗 30 秒就忘了装备。
- **根因**：零件差异在"输出端"（子弹飞行的视觉差异），不在"决策端"（玩家的战术思维不变）。换炮管只改变了子弹轨迹，但玩家的核心循环仍是"走位→瞄准→开火"。
- **影响**：零件系统沦为数值游戏，玩家没有动力探索不同组合。
- **解决方向**：让物理成为核心玩法而非辅助机制。玩家思考"如何让环境杀死敌人"而非"如何打中敌人"。
- **状态**：方案已确定，待实施。

**ISSUE-002: 物理引擎被低度利用**

- **症状**：动量守恒、多米诺链、零摩擦冰面——游戏最独特的系统，目前只有意外触发。
- **根因**：物理击杀没有奖励反馈，玩家不会主动追求环境击杀。
- **状态**：方案已确定（环境倍率+连锁奖励），待实施。

### 🟡 P1 — 玩法深度问题

**ISSUE-003: 围城模式缺乏 tension arc**

- **症状**：6 波从头到尾节奏一致，无起伏，无突发危机，无叙事推进。
- **解决方向**：波次词缀系统（随机变异词缀）+ 地形变化 + Boss 加强。
- **状态**：方案草拟中。

**ISSUE-004: 战斗缺乏 escalation**

- **症状**：第 1 波和第 5 波只有数量的区别，没有"卧槽时刻"。
- **解决方向**：Boss 波 + 突变事件 + 词缀叠加。
- **状态**：部分已实施（Boss + WARNING），待扩展。

**ISSUE-005: 无"build 发现"的探索动力**

- **症状**：玩家没有动力尝试不同零件组合，缺少"发现隐藏 synergies"的快感。
- **解决方向**：零件协同效应系统（特定组合触发隐藏加成）。
- **状态**：方案草拟中。

### 🟢 P2 — 体验/内容问题

## 三、下一步开发方案：环境击杀倍率系统

### 3.1 目标

让物理引擎从"辅助机制"升格为"核心玩法"。玩家主动追求环境击杀，零件选择服务于环境交互策略。

### 3.2 核心设计

| 击杀方式 | 伤害倍率 | 视觉反馈 | 音效 |
|----------|---------|---------|------|
| 直接命中（子弹直击） | ×1.0 | 白色命中粒子 | 标准 hit |
| 反弹命中（至少 1 次墙反弹后命中） | ×1.5 | 蓝色闪光 + 反弹轨迹残影 | 锐利金属音 |
| 方块砸死（被推动的物理方块击中） | ×2.0 | 橙色冲击波 + 方块碎裂粒子 | 重击 bass |
| 多米诺连锁（方块→方块→敌人，≥2 次传递） | ×3.0 | 金色全屏震动 + DOMINO! 大字 | 升调和弦 |
| 反弹+连锁（上述组合） | ×5.0 | 慢动作 + CHAIN KILL 大字 + 全屏金色粒子 | 史诗爆炸 |

### 3.3 实现步骤

1. **Bullet 追踪反弹次数**：在 BulletEntity 中添加 `bounceCount`，moveBullet 中递增
2. **PhysicsBlock 追踪来源**：记录是由哪个坦克推动的，击杀时归属该坦克
3. **DamageModifier 计算**：在 takeDamage 前计算倍率
4. **视觉效果**：每个倍率级别对应不同的粒子/VFX/屏幕震动强度
5. **车间提示**：每种零件的描述中注明"适配击杀方式"

### 3.4 零件与击杀方式的对应

| 零件 | 适配击杀方式 | 理由 |
|------|-------------|------|
| 反射管 | 反弹命中 (×1.5) | 子弹自带反弹 |
| 重型底盘 | 方块砸死 (×2.0) | 可碾碎砖墙 + 推动方块 |
| 烟花炮管 | 多米诺 (×3.0) | 子子弹散射覆盖范围大 |
| 曲射管 | 反弹+连锁 | 抛物线可越过障碍打击后方 |
| 溜冰场地图 | 多米诺 | 零摩擦永恒连锁 |

### 3.5 连锁击杀追踪逻辑

```
击杀时检查伤害来源：
  ├── 来源是 Bullet
  │     ├── bullet.bounceCount > 0 → "反弹命中"  ×1.5
  │     └── bullet.bounceCount = 0 → "直接命中"  ×1.0
  └── 来源是 PhysicsBlock
        ├── block.chainLength = 0 → "方块砸死"  ×2.0
        └── block.chainLength ≥ 1 → "多米诺!"    ×3.0
              └── 如果还有 bounceCount > 0 → "反弹+连锁!" ×5.0
```

## 四、设计哲学

### 核心主张

| 维度 | 过去 | 现在 → 未来 |
|------|------|------------|
| 玩法核心 | 射击 | **物理操纵** |
| 玩家思考 | "我怎么打中他？" | **"我怎么让环境杀死他？"** |
| 零件作用 | 改变子弹外观 | **改变环境交互策略** |
| 独特性 | 有物理引擎的坦克游戏 | **以物理为武器的谜题竞技场** |
| 对标 | 90坦克 / 坦克世界 | **无同类产品 — 新品类** |

### 设计原则

- **操作简单，深度涌现**：方向键+鼠标两键，但环境交互产生无限策略
- **物理即玩法**：反弹、动量、摩擦不是装饰，是核心
- **看见即理解**：倍率用视觉语言（颜色/震动/慢动作）直观传达
