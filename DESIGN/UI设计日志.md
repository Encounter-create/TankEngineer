## 一、当前 UI 问题诊断

| 问题 | 表现 | 影响 |
|------|------|------|
| 无视觉反馈 | 按钮无 hover/active 状态，鼠标悬停无变化 | 用户不知道哪里可以点 |
| 样式不统一 | 大厅/车间/商店/图鉴各有不同配色和字体 | 视觉碎片化，不专业 |
| 文字可读性差 | Canvas 文字小字号模糊，长文本拥挤 | 零件描述阅读困难 |
| 无过渡动画 | 屏幕切换瞬间跳变 | 体验生硬 |
| HUD 信息杂乱 | 血量/弹药/技能/波次全堆角落 | 战斗时分心 |
| 操作提示缺失 | 无快捷键提示 | 新玩家不知道按什么 |

---

## 二、优化方案

### 层次 1：按钮反馈系统（30min）

创建统一按钮组件 `src/ui/Button.ts`：
- `isHovered(mx, my)` 检测
- `render(ctx, mx, my)` 自动绘制 hover/active 状态
- hover 态：边框高亮+背景加深
- active 态：按下时缩放 95%
- 所有 UI 按钮统一调用

### 层次 2：视觉系统统一（1h）

**配色标准化**：

| 用途 | 色值 |
|------|------|
| 主背景 | `#1a1d23` |
| 面板 | `rgba(20,22,30,0.92)` |
| 主按钮 | `#3a5a8c` |
| 危险按钮 | `#6a3a3a` |
| 确认按钮 | `#3a6a3a` |
| 高亮文字 | `#e8e8e8` |
| 次级文字 | `#888` |
| 提示文字 | `#ffcc00` |

**字体层级**：标题22px bold / 正文14px / 辅助11px / HUD 13px bold

### 层次 3：过渡动画 + 快捷键提示（30min）

- 屏幕切换淡入淡出（0.3s）
- 大厅/战斗底部快捷键提示条
