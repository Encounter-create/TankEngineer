从零构建五个"科幻级"指挥官技能，每个都涉及独特的 Canvas 2D 视觉特效和物理机制。开发历时约 8 小时，累计新增代码约 1500 行。

## 1. ☄️ 三体人 — 陨石天降 (Meteor Strike)

### 设计需求
- 按 E 后以鼠标为中心出现红色同心圆 + 白色十字架，闪烁 2 秒
- 巨大火球从左上 45° 加速砸落，带粒子曳尾
- 落地瞬间白屏 + 剧烈震屏，半径 360px 内一切毁灭
- 火圈持续燃烧 10 秒后消散
- 屏幕中央显示"世界属于三体！！！"闪烁大字

### 工程落地
- 状态机 4 阶段：`targeting → incoming → impact → burning`
- 火球从 (-200, -200) 开始，每帧加速 800 px/s² 飞向鼠标位置
- 落地时用 `performance.now()` 记录时间戳，白屏 alpha = `1 - elapsed/5`
- 毁灭逻辑复用已有 `takeDamage(999)` + `block.alive = false`
- 火圈复用 `createFireZone(pos, 360, 10, 30)`
- 十字架撑满 360px 半径圆，8Hz 闪烁

### 困难与解决
| 困难 | 解决 |
|------|------|
| 组装车间无法装备 | `commander_trisolaran` 未加入 `Inventory.ts` 默认库存，加入即可 |
| 演习模式冷却太长 | Practice 中强制 `skillCooldownUntil = now + 1000`（仅演习） |
| 红色大字不消失 | 白屏后才清除 `bivectorText`（后来加了独立字段） |

## 2. 📐 二向箔 — 歌者 (Bivector Foil)

### 设计需求
- 画面从矩形剪切压缩成平行四边形 → 压扁成一条线
- 上边界右移、下边界左移，画面等比例变形
- 压缩过程 12 秒，伴随时 6 句《三体》诗句交替浮现
- 压缩完成瞬间清除一切
- 白屏 8 秒（纯白 2s + 蓝字 3s + 红字 3s）
- 恢复 5 秒缓慢褪去

### 工程落地
- **核心技术：Canvas `ctx.transform(a,b,c,d,e,f)`**
- 变换矩阵 `(1, 0, shear, scale, -shear*cy, cy*(1-scale))` 实现以 Y 轴中点为中心的剪切+压缩
- `shearX` 从 0→0.8，`scaleY` 从 1→0，两者用于 `transform`
- 诗句用 `state.bivectorText` 字段，在 `updateBivector` 中按 2s 间隔切换
- 渲染在 `main.ts` 全局层包裹，不在各模块内

### 困难与解决
| 困难 | 解决 |
|------|------|
| shear=3.0 时画面被推到屏幕外不可见 | 降到 0.8，上边缘偏移 ~192px 正好可见 |
| transform 放在 renderSiege 内无效 | 移到 `main.ts` 的 `render()` 全局层，在 `clearRect` 后立即应用 |
| 白屏 8s + 恢复 5s 完全不执行 | `bv` 变量只判断 `=== 'compressing'`，改了 `!== 'idle'` 但 `?.` 运算符造成 null→true 的陷阱 |
| 演习模式 null 检查失败 | `app.siege?.bivectorPhase` 在 siege=null 时返回 undefined，`undefined !== 'idle'` 为 true，错误把 null 赋给 bv |

## 3. 🐱 薛定谔的猫 — 量子叠加 (Quantum Superposition)

### 设计需求
- 画面红蓝双重曝光交替闪烁 10Hz，震动渐强
- 第 5 秒波函数坍缩，50% 敌人随机瞬间消亡
- 一只像素小猫从左侧跑入 → 中央喵呜 → 右侧跑出
- 坍缩后 3 秒残影消散

### 工程落地
- `updateQuantum` 用 `sin(elapsed*20)` 和 `cos(elapsed*20)` 产生 180° 错相
- 红蓝 alpha：`baseAlpha + 0.2*sin/cos`
- 画面叠加：直接在渲染末尾画半透明红/蓝矩形（不修改已完成的画面）
- 小猫：Canvas Path 绘制（椭圆身体 + 圆头 + 三角耳 + 绿眼 + 胡须 + 四腿 + 尾）
- 猫的位置用段线性缓动：0-4s 左→中，4-5s 停，5-8s 中→右

### 困难与解决
| 困难 | 解决 |
|------|------|
| 猫的 elapsed 在 `collapsed` 阶段计算错误 | 区分 phase：`superposing` 时 `elapsed=5-timer`，`collapsed` 时 `elapsed=5+(3-timer)` |
| 画面重绘两次性能开销 | 不重绘，只在最后叠半透明 tint 层 |
| 猫的行走动画僵硬 | 用 `sin(elapsed*8*2π)` 做身体弹跳 + `sin(elapsed*12*2π)` 做腿部交叉摆动 |

## 4. 🌀 时空曲率 — 爱因斯坦 (Spacetime Curvature Lens)

### 设计需求
- 鼠标位置产生引力透镜，画面径向扭曲
- 内圈蓝移、外圈红移、中心爱因斯坦环
- 敌人螺旋坠入中心，中心压碎
- 2s 形成 → 6s 持续 → 2s 坍缩为奇点

### 工程落地
- **第一版**：同心环 drawImage，每环不同 scale → 用户评价"像放大镜"
- **第二版（最终方案）**：像素级位移映射
  - 画面降采样到 1/3（213×160），`getImageData` 逐像素移位
  - 位移公式：`displacement = strength * (R²/d) * (1-d/R)`
  - 切向扭曲分量 `tangX/Y = ±ndy/ndx * disp * 0.3`
  - 色移：`rMul = 1-ratio*0.5`（蓝移内圈）、`bMul = 0.6+ratio*0.6`（红移外圈）
  - `putImageData` 拉伸回 640×480，浏览器双线性插值平滑
- 物理：径向引力 600 px/s² + 切向力 150 px/s²
- 粒子：蓝红紫色系，密度 4 个/帧

### 困难与解决
| 困难 | 解决 |
|------|------|
| 同心环版本只有均匀缩放，无扭曲感 | 改用 `getImageData` 逐像素位移，每个像素的位移量与到中心距离非线性相关 |
| 全分辨率 30 万像素/帧性能不可接受 | 降采样到 1/3 分辨率（3.4 万像素），`putImageData` 拉伸回全分辨率 |
| 绿色色调像生化武器 | 改紫色基调（`rgba(120,20,180)` → `rgba(80,50,240)`） |
| 内圈绘制覆盖外圈 | 从外向内绘制 + 环形 clip（顺时针外圆 + 逆时针内圆） |

## 5. ⏪ 庞加莱回归 — 时间倒流 (Poincaré Recurrence)

### 设计需求
- 5 秒时间倒流：子弹飞回、敌人退行、粒子倒流
- 蓝色滤镜 + 残影拖尾
- 倒流结束熵爆发：蓝色冲击波毁灭一切

### 工程落地
- **核心思路**：不做真正的状态录制回放，利用现有物理引擎 + 三处控制点
- **控制点 1**：`updateRewind` 在每帧**最前面**反转所有实体速度（只反转一次，`rewindReversed` 标记防振荡）
- **控制点 2**：`handleEnemyAI` 在 rewind 期间 `return` 跳过（防止 AI 覆写反转速度）
- **控制点 3**：子弹制导/烟花分叉在 rewind 期间跳过
- 粒子倒流：`p.life += dt` 生命逆增长，`p.vel *= -1`
- 蓝色滤镜：`rgba(30,60,180, alpha)` 全屏覆盖
- 熵爆：50 个蓝白粒子 + 999 全敌伤害

### 困难与解决
| 困难 | 解决 |
|------|------|
| 技能释放后所有实体静止不动 | `updateRewind` 最初放在循环**末尾**，反转发生在物理/AI 之后，等于无效。移到循环最前面 |
| 实体原地振荡不倒退 | 每帧反转 `vel *= -1` 导致 `+50→-55→+60→-66` 来回摆动。加 `rewindReversed` 标记只反转一次，后续帧加速同向 |
| AI 覆写反转速度 | 反转后的 `enemy.vel` 被 `handleEnemyAI` 立即覆写成正常方向。AI 函数开头加 `if (rewindPhase !== 'idle') return` |
| 摩擦力让速度很快降为零 | 反转后 `moveTank` 的 `FRICTION` 照常衰减速度。每帧强制维持最低速度（敌人 40、子弹 100、方块 30 px/s）+ 持续加速 `scale(1+1.5*dt)` |

## 通用工程经验

### 1. 状态机模式
所有技能都遵循相同的状态机架构：
- `phase: 'idle' | ...` 控制技能生命周期
- `timer: number` 阶段倒计时
- `updateXxx()` 每帧推进状态
- `handleSkillActivation` 触发初始状态

### 2. 渲染层次
- 全局效果（二向箔变形、蓝色滤镜、红蓝双曝）放在 `main.ts` 的 `render()` 中
- 局部效果（陨石红圈、小猫、透镜）在各自模块渲染
- 离屏 Canvas 处理需要捕获+后处理的场景

### 3. 性能策略
- 像素级操作（透镜位移）降采样到 1/3 分辨率
- 画面叠加（量子双曝）只用半透明 tint 层，不重绘
- 变形变换（二向箔）用 Canvas 原生 `transform`，零额外开销

### 4. 粒子系统
- 所有技能复用 `state.particles` 数组
- 粒子类型通过 `color` 区分，不做新数据结构
- 每帧随机生成，用 `Math.random() < threshold` 控制密度

### 5. 演习模式一致性
- 所有 `updateXxx()` 函数 export，Practice.ts import 后直接调用
- PracticeState 镜像 SiegeState 的技能字段
- 每次新增字段需同步三处：SiegeState 接口、init、PracticeState 接口、Practice init

### 6. 库存管理
- 新指挥官加入 `MVP_COMMANDERS` 数组（Parts.ts）
- 新指挥官加入 `defaultInventory`（Inventory.ts）
- `DATA_VERSION` 每次新增零件时递增

## 8. 🌊 诺亚方舟 (Noah's Ark)

### 设计需求
- 大洪水：深蓝水面从底部攀升（Canvas clip）
- 暴雨粒子斜向密集下落 + 金色闪电 + 白屏闪 + 震屏
- 水面"黏性"：接触到的一切（敌军/物块）被锁死在水面上
- 涨潮时随水上升，落潮时被卷出地图边界销毁
- 方舟（玩家所在）漂浮于水面上
- 11s 涨潮 + 1s 峰值 + 11s 落潮 = 23s 总长

### 工程落地
- 水位计算：`MAX_H * progress²`（二次缓入缓出）
- 三层波浪叠加渲染：深浅快慢不同 sin 波 + 波峰高光线 + 浪花圆点
- 黏性表面：`enemy.pos = Vec2(sx, waterY - R)` + `enemy.vel = Vec2.zero()`（冻结）
- 物体随水移动：`sx = pos.x + 250*dt`（涨潮退潮统一速度）
- 方舟绘制：极简弧形船体 + 矩形船舱（~15 行）
- 暴雨：4 条/帧雨滴粒子 + 渲染层 15 条雨线（clip 到 waterY 以上）
- 闪电：2-5 段随机折线 + 白光 flash + 金色粒子 + 震屏

### 困难与解决
| 困难 | 解决 |
|------|------|
| flooding 阶段水面从 MAX_H 骤降至 0.3×MAX_H | flooding 直接设 `MAX_H` 不变，无缝衔接 raining 终点 |
| 落潮时物块堆积在右下角不消失 | 边界检查 `MAP_W+30` 被物理碰撞回弹 → 改为滑到 `MAP_W-20` 直接 `block.alive = false` |
| 黏性表面无效，物体自由沉入水底 | 直接设位置 + `vel = Vec2.zero()`，AI 无法覆盖 |
| 雨水只有半边天 | 雨线加 `clip(0, 0, MAP_W, waterY)`，暴雨区+洪水区互补全屏 |
| 波浪看起来像静止的锯齿 | 时间系数 ×3 加速，振幅 ×2 增大波峰波谷 |
| 水面骤升骤降 | `progress²` 二次缓动曲线替代线性 |
| 单层蓝色渐变像"蓝布" | 三层波浪叠加（深浅快慢）+ 白蓝浪花粒子 + 水体内部湍流粒子 |

## 9. ⚔️ 达摩克利斯之剑 (Sword of Damocles)

### 设计需求
- 每个敌人头顶悬下巨剑 + 细线
- 4s 悬停(白线) → 1s 加速下落(红线) → 剑中腹命中敌人中心
- 敌人越动越慢（感受到剑的重量）
- 命中瞬间：神圣金光粒子爆发（非爆炸式）
- 随机台词：3 套经典语录逐行展示

### 工程落地
- 剑绘制 (~50 行)：锥形双刃剑身+中脊线+血槽，翼型护手(红宝石)，皮革缠绕握柄，金球红宝石配重球
- 下降物理：`descend = 138 * t²`（`d = ½at²`），t = `1 - timer/0.707`，t=1 时剑腹命中 `ey`
- 锚点定位：`ey - TANK_RADIUS * 13 - 5`（12 个坦克身高），下降距离 138px
- 粒子系统：金柄粒子（`#fff8c0/#ffdd44`）+ 冷蓝剑身粒子（`#4488ff/#3355cc`）+ 金色曳尾 + 60 粒子神圣击杀

### 困难与解决
| 困难 | 解决 |
|------|------|
| 剑尖碰到敌人就消失（不是剑腹）| 下降距离从 112→138（半剑长偏移 ~25px） |
| 下落距离被改后剑柄穿过敌人 | 回到只改锚点不改距离的思路，精确计算 descend = 锚距 - 剑腹偏移 |
| 每一帧都在闪（双 reversal）| 加 `rewindReversed` 标记，只反转一次，后续帧加速同向 |
| 台词被技能结束截断 | 创建独立 `QuotePlayer` 模块，单例模式，与技能层完全解耦 |
| 台词切换时闪烁 | `alpha = 0` → `alpha = 0.05`，新行立刻有微弱可见度 |
| 蓝光粒子颜色太浅看不到 | 纯蓝系 `#4488ff/#3355cc` 替代蓝白系 |
| 爆炸像炸药不是神圣感 | 60 个细金粒子慢扩散 + smokeExpand → 金色光芒取代爆炸 |

## 10. 🎬 QuotePlayer — 独立台词系统

### 设计动机
技能的机制层（战斗效果）和台词层（文字展示）必须解耦。台词不能因为技能结束而被截断。

### 架构
```
QuotePlayer (模块级单例)
├── playQuote(lines)    ← 技能 handler 一句调用
├── updateQuote(dt)     ← main.ts update 中独立运行
└── renderQuote(ctx)    ← main.ts render 末尾独立渲染
```

### 核心设计
- 模块级 `state` 不暴露给外部，技能层不需要持有任何台词状态
- 逐行淡入（2s/行，金色 `rgba(255,220,100)`）
- 全部显示后 5s 整体淡出
- 默认模式，以后所有文学/文化类技能统一使用

## 核心经验总结

### 7. 典故优先于机制
机制是手段，故事才是目的。先找典故 → 把故事翻译成游戏事件 → 特效讲好这个故事。玩家按 E 键的"卧槽"瞬间比任何机制都珍贵。

### 8. 台词系统解耦
台词和技能必须独立运行。技能负责战斗，台词负责叙事，两者通过一句 `playQuote(lines)` 信号关联。技能结束不影响台词播放。

### 9. 视觉即故事
每个粒子都要"会说话"。木马的金光 = 希腊联军伏兵的紧张感。方舟的浪花 = 大洪水的狂暴。神圣击杀的细金粒子 = 达摩克利斯之剑的威严，不是爆炸。

### 10. 参数调优的黄金法则
- 所有距离计算都要从锚点精确推导到目标位置（不靠猜）
- 每次修改都要追溯公式，保证 `t=1` 时精确命中
- 粒子颜色不能太浅（用户看不到），不能太像烟雾（不像光芒）
