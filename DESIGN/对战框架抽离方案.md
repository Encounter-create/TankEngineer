# 对战框架抽离方案

## 核心理念

围城模式 = 对战框架 + 基地 + 波次 + 计时。

对战框架是标准化的 update 管道 + render 管道，任何新模式通过"框架 + 新模式特有逻辑"组装得到。

组合优于继承。

## 当前问题

Practice.ts 和 Siege.ts 的 update/render 循环是两套独立代码，循环结构重复。每次加新功能都要改两个文件。

## 通用框架（约 70%）

```
updateBattle:
  ├─ Player input + fire
  ├─ Enemy AI
  ├─ Allies / Turrets / Planes / Clones
  ├─ Bullets + bullet-tank collisions
  ├─ Physics blocks + collisions
  ├─ Particles + damage numbers
  ├─ Fire zones
  ├─ Skill updates (all 10+ skills)
  └─ Screen shake decay
```

## 模式特有（约 30%）

### 围城模式
- 指挥所攻击 + 防御范围
- 波次生成
- 波次修饰符
- 3 分钟倒计时
- 胜利/失败判定

### 演习模式
- 固定靶（无敌、不可移动）
- 活动靶（AI 敌人）
- 重置按钮
- Boss 召唤
- 调试模式
- 1s 技能冷却覆盖

## 工程方案

```typescript
// src/core/BattleEngine.ts
export function updateBattle(state: BattleState, input: Input, dt: number): void {
  updatePlayerInput(state, input, dt);
  updatePlayerFire(state, input, dt);
  handleEnemyAI(state, dt);
  handleAllies(state, dt);
  handleTurrets(state, dt);
  handlePlanes(state, dt);
  handleClones(state, dt);
  handleBullets(state, dt);
  handleBulletTankCollisions(state, dt);
  handlePhysicsBlocks(state, dt);
  handleFireZones(state, dt);
  updateParticles(state, dt);
  updateDamageNumbers(state, dt);
  updateSkills(state, dt);
  decayScreenShake(state, dt);
}

// src/modes/Siege.ts
function updateSiege(state: SiegeState, input: Input, dt: number): void {
  updateBattle(state, input, dt);
  handleCCAttack(state, dt);
  handleWaves(state, dt);
  handleMatchTimer(state, dt);
  applyWaveModifiers(state);
}

// src/systems/Practice.ts
function updatePractice(state: PracticeState, input: Input, dt: number): void {
  updateBattle(state, input, dt);
  // 演习特有逻辑
}
```

## BattleState 接口

```typescript
interface BattleState {
  player: TankEntity;
  enemies: TankEntity[];
  bullets: BulletEntity[];
  physicsBlocks: PhysicsBlock[];
  particles: Particle[];
  fireZones: FireZone[];
  damageNumbers: DamageNumber[];
  allies: AllyTank[];
  clones: CloneEntity[];
  turrets: TurretEntity[];
  planes: Plane[];
  aiContexts: Map<string, AIContext>;
  map: TileGrid;
  screenShake: number;
  // skill fields...
}
```

SiegeState 和 PracticeState 都 extends BattleState。

## 收益

| 收益 | 说明 |
|------|------|
| 零重复 | 通用逻辑只写一次，新模式只需追加特有代码 |
| 易扩展 | PvP、Boss 战、生存模式直接框架+配置 |
| 易维护 | 修一个 bug，所有模式同步受益 |
| 类型安全 | BattleState 统一接口，不会漏字段 |

## 风险

- SiegeState 和 PracticeState 的字段差异需要统一（如 PracticeState 没有 `aiContexts`）
- 需要确保 renderBattle 同样抽离（渲染管道也有大量重复）
- 建议和 render 一起做，先 update 后 render

## 实现时机

建议在下一个新游戏模式开发之前完成此重构。目前 Siege.ts 已从 2850 行拆至 1724 行，再做这一层拆分可进一步降至 ~800 行。
