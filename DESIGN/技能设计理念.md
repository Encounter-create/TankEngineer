# 坦克工程师 — 技能设计理念

## 核心洞见

**机制是手段，故事才是目的。**

市面上不缺酷炫的技能特效。但「用典」和「玩梗」——把文学、历史、科学进行二次艺术加工，让玩家按 E 键的瞬间产生"卧槽，这是 XX！"的会心一笑——这件事极少有游戏在做。

这是我们的差异化赛道：**以游戏为载体，做文化创新。**

---

## 设计原则

### 1. 典故优先于机制

不要先想"做个召唤技能"，要先想"有没有哪个典故讲召唤的"。找到典故后，机制自然从故事中生长出来。

> ❌ "做个闪电链" → ✅ "庞加莱回归——时间倒流，一切回到初始状态"

### 2. 玩家三秒内必须 get 到

技能从释放到让玩家理解典故，窗口期不超过 3 秒。不要设计需要阅读说明书才能理解的技能。

> ✅ 特洛伊木马：木马滑入→开门→出人 → 瞬间代入荷马史诗
> ❌ 太复杂的逻辑链 → 玩家只觉得"这技能怪怪的"

### 3. 视觉即故事

特效不只是"好看"，而是"把故事画出来"。一个粒子效果如果不能传达叙事，就是噪音。

> 木马的金光粒子 = "希腊联军在木马中等待出击的紧张感"
> 方舟的暴雨+闪电+黏性水面 = "大洪水淹没一切的绝望与希望"

### 4. 幽默是最好的记忆锚点

严肃的技能大家都忘，搞笑的技能大家记一辈子。反差萌、玩梗、彩蛋文字，成本极低但记忆价值极高。

> 木马骑手头顶写"Odysseus" + 名言彩蛋
> 萌龙抱紧敌人 + 爱心粒子

### 5. 简单直接 > 复杂炫技

技能逻辑越短越好。玩家按下 E，一眼看清发生了什么。二向箔就是"画面压扁"，三体人就是"大火球砸下来"。

## 技能分类矩阵

| 类型 | 举例 | 创作方法 |
|------|------|---------|
| **文学典故** | 二向箔（《三体》）、特洛伊木马（《伊利亚特》）、诺亚方舟（《圣经》） | 把经典情节翻译为游戏事件 |
| **科学概念** | 薛定谔的猫、庞加莱回归、时空曲率 | 把抽象概念具象化为视觉+物理效果 |
| **历史文化** | 长城烽火、草船借箭、空城计 | 把著名场景变成游戏互动 |
| **玩梗搞笑** | 叶公好龙 | 反差萌 + 幽默粒子 |

## 台词系统

- 文学/文化/历史/神话类技能**必须配台词**，从原文中选取最经典语录
- 每技能准备**多套台词随机选**（彩蛋效果）
- 默认展示模式：每行 2s 淡入 + 金色字样 + 全部显示后 5s 整体淡出
- 实现：`QuotePlayer` 独立模块，与技能层完全解耦，一句 `playQuote(lines)` 调用

## 技能评价标准（我们自己的 checklist）

每设计一个技能，问自己：

1. 典故/梗是什么？玩家能认出来吗？
2. 释放后 3 秒内玩家能理解发生了什么吗？
3. 视觉特效在"讲故事"还是在"堆特效"？
4. 有没有一个让玩家微笑/眼前一亮的瞬间？
5. 逻辑链够不够短？（>3 步可能就要砍）
6. 台词配了吗？（文学/文化/历史/神话类必须）

## 开发模式

```
1. 先讲典故 → 用户确认"这个梗我 get 到了"
2. 拆分时间线 → 分阶段画故事板
3. 确定特效语言 → 每个粒子、每种颜色都要"会说话"
4. 落地实现 → 复用已有模式（状态机 + updateXxx + drawXxx）
5. 迭代微调 → 开游戏实测，感受"那个瞬间"是否够震撼
```
